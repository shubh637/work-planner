DESIGN NOTES
------------
  1. Spring Security returns HTTP 403 (not 401) for unauthenticated
     requests when no token is provided (default framework behavior).

  2. HTTP status codes used by the API:
     - 200 OK          : successful updates/reads
     - 201 Created     : successful resource creation
     - 204 No Content  : successful delete
     - 400 Bad Request : validation errors, duplicate data
     - 403 Forbidden   : role-based access denial
     - 404 Not Found   : resource not found
     - 422 Unprocessable Entity : invalid state transitions
       (approve already-approved, task on DONE project, etc.)

  3. PUT /users is restricted to MANAGER role only. Members must
     use dedicated endpoints to update their own profile.

  4. DONE project enforcement: The backend blocks task creation,
     suggestion, and assignment on projects marked DONE.
